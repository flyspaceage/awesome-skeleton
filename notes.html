The Basics of Programming

How Does The Internet Work?

The internet is an interconnected network of computers that communicate with one another using Hyper Text Transfer Protocol, or hyperlinks for short. Each time a web page is visited, the computer sends an HTTP Request to a server, a computer specifically designed to receive and send HTTP requests. The server responds to the computer by sorting through 30 billion HTML documents, selecting the correct document and sending the information back to the computer which displays the document on a web browser, like Chrome or Firefox.

What is HTML?

HTML stand for Hyper Text Markup Language. HTML documents are the core of of the internet and make up almost everything on the web. HTML documents contain the text, images, and videos i.e. text content, as well as the format, i.e. markup, of a web page.

How to Structure an HTML Document

Every time you begin writing a new coding document there are some basic pieces of information you need to include, such as the type of document and the coding language. HTML5 simplifies much of this process, but programmers must still differentiate between the head and body of an HTML document. The head will include the title of the document as well as any styling information, i.e. CSS and JavaScript. The body will include all of the content, i.e. text, images, videos.

What are Tags, Elements, and Attributes?

HTML documents consist of elements that are organized by tags. Each element has an opening and closing tag with content in between. Elements may be nested within eachother but it is best to close out elements as soon as they are finished. Some tags also use attributes to further define values assigned to an element. Attributes provide additional information about the elements and are always contained within the opening tag. Watch this video to review the basics of elements and tags. Some examples follow below:

  1. Tag
  2. <a> - used for links
  3. <b> - style as bold
  4. <em> - style as italics
  5. <p> - represents a paragraph of text
  6. <br> - line break
  1. Attributes
  2. <href> - used for links
  3. <src> - source of image
  4. <alt> - alternate text
  5. <lang> - language

Explain the Difference Between Inline vs. Block Elements

Every HTML element has a default display value, either inline or block. An inline element takes up only as much width as the text it contains. A block element always starts a new line and takes up the entire width of the line. By using attributes such as style and class some design values may be assigned to elements within the body of the HTML document. Some examples follow below:

  • Block Elements
  • <div> - basic container
  • <p> - paragraph
  • <form> - menu/drop down list
  • <h1-h6> - headers
  • <address> - contact information
  • Inline Elements
  • <a> - hyperlink
  • <span> - container
  • <br> - page break
  • <img> - image
  • <strong> - bold

Creating a Structured HTML Document

What is the purpose of HTML, CSS, and JS?

HTML, CSS, and JavaScript are unique languages with their own syntax and rules that allow web developers to build basic strucutre (HTML), implement design (CSS), and add interactive components (JS) to web pages. HTML controls the structure of a web page while CSS controls the style, or how the web page looks. HTML elements are either inline or block. Block elements form an invisible box around the content. Cascading Style Sheets (CSS) use syntax and rules to change the style of the boxes, such as font size, color, background, border, etc. CSS allows programmers to assign attributes and values to HTML elements. In the example below, the tag is blue, the attribute is red, and the value is green. HTML elements may have multiple classes.

<h1 class="content">text<h1>
<tag attribute="value">text<tag>

Each value corresponds to a design element that can be applied universally to avoid repitition. To tie structure and style together, the HTML document provides the browser with information about what style to use and where to get it from (style sheet). Front End web developers use integrated web tools like Developer Tools in Chrome to gain insight and immediate feedback about how a web pages HTML and CSS are structured and/or designed.

Explain the Document Object Model

The Document Object Model is a universal interface for platforms and languages that allows programs and scripts to dynamically access and update the content, structure, and style of documents. When programmers talk about DOM aka Document Object Model, they are referring to the tree structure of a web page. When reading an HTML document as text the undulating indentations indicate where an object is located. The further an object is indented the more deeply nested its corresponding box. In a web browser, this structure shows up as a series of boxes within boxes. To learn more about this concept check out this link.

Boxes on Boxes on Boxes

The design philosophy behind programming revolves around the idea of boxes. Boxes make up everything on a web page and programmers must visualize this concept in order to implement the content and style requested. To apply style to one of these boxes, simply assign an element a class attribute, which is like the label on the box. HTML tags may have multiple classes. When approaching a new design always start with largest element (the web page) and work towards the smallest. When in doubt, sketch it out!

Creating a Cascading Style Sheet

Understanding the Box Model

Cascading, or the most specific rule applied, hints at the golden rule of CSS - do NOT repeat yourself. Multiple CSS sheets may be used to determine the final design of an HTML document. CSS relies on selectors to create multi-faceted designs and a wide range of styles. Each element is a box and two selectors may be used to define an element. In CSS, each element has four core adjustable parts (seen below): content, padding, border, and margin. Content and padding are affected by the background color of an element while border inherits color but margin remains translucent.

Avoid Repition at ALL Costs!

Don't Repeat Yourself (DRY). CSS allows programmers to determine the style of a webpage while practicing the ethos of avoiding repitition. Not repeating code enhances the maintainability of a webpage, making it simple and easy to make future changes or allow a new programmer to step in and pick up where another left off. More so, reducing repetition allows for code to function across applications and be re-used more readily. Check out this article for a hands-on walk-thru highlighting the value of DRY.

To abolish repetition, programmers must understand the concept of inheritance. Inheritance is a key feature of CSS. Inheritance relies on the document tree, which is the hierarchy of HTML elements in a page based on nesting. Descendant, or child, elements may inherit CSS property values from any ancestor, or parent, element enclosing them. Generally, child elements inherit text-related properties, but not box-related properties. Properties that can be inherited are color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, whitespace, and word-spacing. Properties that cannot be inherited are background, border, display, float, clear, height, width, margin, minimum and maximum height, position, and align. Learn more about inheritance here.
